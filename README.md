# weather-agent — エージェントループ最小実装

> RAYVEN 技術課題「Growth Finder AI — 1on1 フィードバック生成エージェント」の
> 前段階として、**エージェントループの本質だけ**を最小構成で検証した素振りリポジトリ。

## これは何か

「大阪、明日は何を着ればいい？」という問いに、LLM が**自らツールを呼んで天気を取得し**、
その事実にもとづいて服装を提案する最小エージェント。ターミナルで動く 1 ファイル（`agent.ts`）。

本番課題の心臓部は、LLM を 1 回呼ぶだけの API ではなく、
LLM の判断とツール実行を反復する**エージェントループ**にある。
新しい要素が多い本番にいきなり着手する前に、このループだけを裸で動かして本質を掴むために作成した。

## なぜ「天気 → 服装」という題材か

LLM 単体では答えられない問いだから。LLM は明日の天気を知らないため、
一般論に逃げるか、もっともらしい嘘を作るしかない。
だから **LLM 自身が「天気を取りに行く必要がある」と判断してツールを呼ぶ**しかなく、
エージェントループの必然性がそのまま体験できる。

これは本番課題の「承認するには過去評価との比較が必要 → だから履歴取得ツールを呼ぶ」と
同じ構造になっている。

## 動かし方

```bash
npm install
echo 'ANTHROPIC_API_KEY=sk-ant-...' > .env
npx tsx --env-file=.env agent.ts
```

## 実装のポイント

- **エージェントループ**: `for` ループで LLM を呼び、`stop_reason` を見て分岐する。
  `tool_use` ならツールを実行し結果を会話履歴へ追加して再度呼ぶ。それ以外なら最終回答としてループを抜ける。
- **maxIterations によるループ上限**: ツール呼び出しの暴走・無限ループを防ぐ（上限 5 回）。
- **ツールは read-only**: 天気を取得するだけで書き込みはしない。
- **ツールレジストリ**: ツール名 → 実行関数の対応表を 1 か所に定義し、
  名前をハードコードで分岐しない。命名の揺れによる不一致バグを構造的に防ぐ。
- **失敗を握りつぶさない**: 未知のツール名を要求された場合も例外で落とさず、
  `is_error` を立てた tool_result として LLM に返す。LLM はそれを踏まえて自己修正できる。

## 本番課題との対応

このリポジトリで検証したループの骨格は、本番課題とそのまま対応する。
本番は「中身の差し替え」であり、骨格は同一。

| 素振り（本リポジトリ）              | 本番課題（Growth Finder AI）                                          |
| ----------------------------------- | --------------------------------------------------------------------- |
| `for` ループ + `stop_reason` 分岐   | エージェントループ                                                    |
| `getWeather()`（ダミー, read-only） | `getPreviousEvaluation` / `getEvaluationTrend`（Supabase, read-only） |
| `tools` スキーマ                    | `tools.ts`                                                            |
| ツールレジストリ                    | ツール実行のディスパッチ                                              |
| `is_error` フラグ                   | `ToolResult` ユニオン型                                               |
| `maxIterations = 5`                 | `maxIterations = 5`                                                   |
| `console.log("[ツール実行]...")`    | `trace.ts`（実行ログ・コスト記録）                                    |
| ユーザーの質問文                    | `prompt.ts` + 店長の QHC 評価入力                                     |
